# Текущий этап разработки

## Сделано
- Реализована часть бизнес-логики для магазинов:
  - Импорт каталога магазина из YAML-файла
  - Просмотр подзаказов магазина (все, кроме статуса "корзина"): адрес доставки, статус заказа/подзаказа, позиции
  - Управление активностью магазина (включение/отключение приёма заказов)
  - Управление профилем магазина (категории, название, урл)
  - Редактирование позиций (добавление/удаление/изменение)
  - Смена статуса подзаказа (ShopOrder)
  - Просмотр профиля своего магазина
- Рефакторинг моделей:
  - Добавлена модель ShopOrder для разбиения общего заказа на подзаказы по магазинам
  - У подзаказов реализован собственный статус
  - Product сделан моделью "эталонов" товаров (добавлено уникальное поле model — код/семейство товара)
  - ProductInfo сделан моделью офферов конкретного магазина (убрано поле name, добавлено external_id)
- Пересмотр эндпоинтов

## В процессе
- Переписывание и финализация сериализаторов
- Настройка реальной отправки писем (вместо console backend)
- Отправление накладных администратору (реализация внутри бизнес-логики клиента)
- Бизнес-логика клиента
- Переработка ChangeOrderStatus - дать пользователю возможность обновлять статус на canceled вне жесткой очереди статусов

## Планируется
- Реализация бизнес-логики для пользователя-покупателя (`type: buyer`)
  - Просмотр каталога (с фильтрацией/сортировкой)
  - Корзина: добавление/удаление товаров
  - Адреса доставки: добавление/редактирование, выбор адреса по умолчанию
  - Оформление заказа на выбранный адрес (если не указан — использовать дефолтный)
  - Просмотр списка своих заказов
  - Отмена заказа (после "корзина", но до "доставлен")
  - Смена пароля
- Возможный рефакторинг моделей
